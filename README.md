# Interval Forecasting Pipeline

Production-style, multi-stage forecasting pipeline for 30-minute interval
contact-center metrics — **Call Volume (CV)**, **Customer Care Time (CCT)**, and
**Abandoned Rate (ABD)** — across four portfolios (A, B, C, D). Designed as a
reusable internal analytics tool: a Python package for modeling, CLI entrypoints
for one-command runs, and a React dashboard for stakeholder review.

**Live dashboard:** [predicting-cc.vercel.app](https://predicting-cc.vercel.app/)

```

│  Excel + API │   │ calendar/lags │   │ Stage1 → Stage2│   │ CSV + JSON  │
```

## Modeling approach

**Stage 1 — daily level.** Pooled gradient-boosted models predict *daily*
portfolio totals with loss functions matched to each target and a downside-heavy
scoring asymmetry:

| Metric | Objective | Notes |
| --- | --- | --- |
| Call Volume  | `reg:quantileerror` | per-portfolio α ∈ [0.55, 0.62] biases predictions upward |
| Care Time    | `reg:quantileerror` @ α = 0.57 | same upward bias on per-call handle time |
| Abandoned    | `count:poisson` | modeled as counts; rate derived post-hoc as `abd_calls / CV` |

**Stage 2 — interval shape.** Daily totals are disaggregated to 48 × 30-minute
intervals using learned **day-of-week × half-hour** shape profiles calibrated on
Apr–Jun 2025 interval observations.

**Calibration.** Isotonic calibration on a leave-month-out holdout to correct
systematic bias. A seasonal-naive blend is applied to Abandoned Calls with
per-portfolio weights (A=0.75, B=0.65, C=0.95, D=0.85).

**External enrichment.** US federal holidays fetched from the public
[nager.at](https://date.nager.at) API; responses cached locally to
`data/interim/holidays.json` for reproducible offline runs.

## Quickstart

```bash
make install          # Python deps + dashboard npm deps
make pipeline         # ingest → features → fit → forecast → exports/
make backtest         # leave-month-out with asymmetric MAPE
make dashboard        # launch React dashboard on :5173
make test             # pytest
```

Single-command deploys and scheduled re-runs: the `scripts/` CLIs are
`click`-based and composable with cron, Airflow, or any standard scheduler.

## Layout

```
pipeline/            # importable Python package — source of truth
  config.py          # paths, portfolios, hyperparameters, loss weights
  ingest/            # Excel loader, nager.at holidays client (cached)
  features/          # calendar, lag/rolling, DOW × half-hour profiles
  models/            # Stage1Daily, Stage2Shape, isotonic calibrator
  evaluate/          # asymmetric MAE/MAPE, leave-month-out backtest
  export/            # submission CSV + dashboard JSON writers
scripts/             # CLI entrypoints (run_pipeline, run_backtest, build_dashboard_payload)
dashboard/           # Vite + React + Recharts (deploys to Vercel)
data/
  raw/               # input Excel workbook + reporting template
  interim/           # cached API responses, intermediate frames
  processed/         # model-ready feature frames
  exports/           # forecast CSV + dashboard payload JSON
research/            # exploratory notebooks (frozen for traceability)
tests/               # pytest unit tests
```

## Dashboard

Vite + React 18 + Recharts. Reads `/dashboard.json` generated by the export
stage — no backend required. Deployable to Vercel via the `vercel.json` at
repo root, or statically to any CDN.

Surfaced panels:
- KPI tiles (total CV, peak interval, avg CCT, abandoned rate, busiest day)
- Intraday 30-min forecast chart per portfolio × metric × day
- Daily totals table
- Portfolio comparison overlay
- Day × hour heatmap across the forecast month
- Live view of model configuration (quantile α, loss weights, blend weights, calibration strategy)

## Reproducibility

- Random seeds fixed in `pipeline/config.py`.
- Holiday cache committed under `data/interim/` — offline-safe.
- `pyproject.toml` pins minimum versions; lockfile generated via `pip-compile`.
- Research notebooks under `research/` document the analytical path; the
  `pipeline/` package is authoritative for production runs.

## Asymmetric scoring

The evaluation harness penalizes underprediction more than overprediction
(default `under × 1.5` / `over × 1.0`, configurable in `pipeline/config.py`).
Asymmetric MAPE is reported per metric in `scripts/run_backtest.py`.
